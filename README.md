# 诗外星辰 (Verses & Voids) V5.5 极致交互与 AI 创意版

> **“以星辰之势，重绘千年文人星图。”**

`诗外星辰` 是一个基于 **多巴胺波普 (Dopamine Pop)** 美学设计的 3D 交互式文人关系时空观测站。本项目利用 3D 引力图可视化技术，将散落在浩瀚史料中的诗人关系（从盛唐到大清）重构为动态的社交星云，旨在为古典文学的研究与欣赏提供一个充满活力、极具现代感的视角。

---

## 💎 项目定位与价值

### 1. 核心定位

不同于传统的静态图谱，`诗外星辰` 致力于构建一个 **“活着的文学宇宙”**。它不仅是数据的可视化，更是一个可以交互、可以感知的时空剧场。

### 2. 核心价值

- **文化传播**：通过潮流的 3D 视觉语言，打破古典文学的沉闷感，吸引年轻群体探索传统文化。
- **关联研究**：辅助研究者直观观察不同朝代、不同流派（如苏门四学士、蜀党、变法派）的社群拓扑结构。
- **沉浸教育**：通过极简的操作与直观的视觉反馈，显著降低了理解古代复杂人际网的门槛。

## 📊 数据逻辑与采集处理

### 1. 数据来源

项目数据依托于以下公开资源进行深度挖掘：

- **古诗文网**：采集诗人基础生平、传世作品数及名篇内容。
- **维基文库/全宋词/全唐诗**：提取诗人间的赠答、唱和、引用关系。
- **中国哲学书电子化计划**：校验人物生卒年及地理迁徙轨迹（Spatiotemporal data）。

## ✨ 核心功能 [NEW]

### 1. 诗人星系图谱 (Poet Stargraph)

- **动态关系网**：通过 3D 引力布局展示唐代诗人之间的社交、师承与地域联系。
- **星系导出**：支持将当前的星系布局导出为极具艺术感的精美卡片。

### 2. 全方位 AI 创意引擎 [V5.5]

- **AI 深度赏析 (AI Appreciation)**：针对特定诗句生成意境、技巧与背景的深度剖析，采用“纸质卷轴”流式输出。
- **AI 模仿作诗 (AI Imitate Poem)**：基于特定诗人风格，输入主题即可生成富有韵味的临摹之作。
- **AI 跨时空对话 (AI Chat)**：开启不同朝代文豪间的“圆桌对话”，探讨共同命题。
- **飞花令对决 (AI Poetry Game)**：与 AI 文豪进行飞花令互动，验证您的诗词积累。
- **AI 画境工坊 (AI Scene Director)**：解构诗句意境，生成专业的 AI 绘画描述词（Prompt）。

### 3. 极致交互体验

- **顶部集成搜索**：搜索框全面整合至顶部标题栏，支持跨屏自适应。
- **时空回溯滑块**：动态过滤星系中的诗人，观察文坛权力谱系的演变过程。
- **移动端深度适配**：完美解决输入法自动缩放 (16px 规范) 与视口溢出问题。

## 🛠️ 配置与部署

项目支持两种方式配置 AI 创意引擎：

### 1. 网页端 UI 配置 (推荐)

点击页面右上角的 **“引擎配置” (Gear 图标)**，即可填入您的 API 配置：

- **API Key**: 来源 API 的访问密钥。
- **Base URL**: API 接口地址（如 `https://api.openai.com/v1`）。
- **Model**: 指定使用的模型名称。

*注：此配置仅保存在您的浏览器本地缓存中。*

### 2. 服务器环境变量配置 (安全生产)

在部署到服务器或本地开发时，可以在 `.env.local` 文件中预设以下变量：

```bash
AI_API_KEY=您的密钥
AI_BASE_URL=API基础路径
AI_MODEL=模型名称
```

系统会优先使用环境变量。如果未配置，则会自动提示用户通过 UI 填写。

## 🎨 视觉架构设计

### 1. 节点设计 (Nodes)

- **层级视觉**：节点半径与影响力（val）呈对数增长关系，顶级诗人拥有显著的内核。
- **光晕系统**：节点外围包裹着一层根据朝代颜色渲染的半透明光晕。

### 2. 连线设计 (Links)

- **社交能量场**：利用粒子流动 (Directional Particles) 表现社交互动的强度与方向。
- **星辰化 (Stellarization)**：一键切换“同时代”感知连线，还原真实的历史背景。

## 🚀 性能优化与技术方案

- **纹理智能缓存 (Texture Caching)**：针对高密度的 3D 文本标签，通过 Canvas 纹理缓存极大地降低了 GPU 负担。
- **自适应渲染机制**：移动端自动降低粒子密度并启用简化的节点效果，确保流畅操作。
- **Viewport 安全区**：通过 `calc(100vw - 2.5rem)` 宽度缓冲区，确保装饰阴影不触发视口扩张。
